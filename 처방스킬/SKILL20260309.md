## 1. 왜 이 스킬이 필요한가?

화장품 제조 현장에서는 **단일 INCI 성분** 대신 **복합성분 블렌드(Compound/Blend)** 원료를 투입한다.

```
[문제 상황]

처방서 투입 기준 (제조용)          전성분 표기 기준 (규제/소비자용)
─────────────────────────────    ──────────────────────────────────
Bentone Gel MIO        5.00%  →  Cyclopentasiloxane        4.25%
                               →  Disteardimonium Hectorite  0.50%
                               →  Propylene Carbonate        0.25%

Cyclopentasiloxane    15.00%  →  Cyclopentasiloxane       15.00%
                                                           ────────
                                  최종 전성분 표기          19.25%  ← 합산 필수!
```

> [!WARNING]
> 복합성분 전개 없이 처방서를 그대로 전성분으로 출력하면 **동일 INCI 중복·누락 오류** 발생

---

## 2. 핵심 처리 흐름

```
Step 1: 처방서 로드 (투입 원료 기준)
        ↓
Step 2: 복합성분 감지 (DB 조회 또는 TDS 기반)
        ↓
Step 3: 복합성분 전개 — 구성 INCI × 투입량 계산
        ↓
Step 4: INCI 합산 — 동일 INCI 중복 처리
        ↓
Step 5: PRECISION-ARITHMETIC 연계 — 정수 변환 + 역산 + 3단계 검증
        ↓
Step 6: 전성분 출력 (표기 기준)
```

---

## 3. 복합성분 전개 계산 규칙

### 규칙 A — 전개 공식

```
구성_INCI_wt% = 복합원료_투입wt% × 구성_INCI_비율(fraction)
```

**예시 — Bentone Gel MIO 5.00% 투입 시**

| 구성 INCI | 비율 (fraction) | 전개 wt% |
|---|---|---|
| Cyclopentasiloxane | 0.850 | 5.00 × 0.850 = **4.25%** |
| Disteardimonium Hectorite | 0.100 | 5.00 × 0.100 = **0.50%** |
| Propylene Carbonate | 0.050 | 5.00 × 0.050 = **0.25%** |
| **소계** | **1.000** | **5.00%** ✅ |

> [!IMPORTANT]
> 구성 INCI 비율의 합계는 반드시 **1.000 (100%)** 이어야 한다.  
> TDS에 비율이 없을 경우 → Gemini 리서치 트리거 (§8 참고)

### 규칙 B — INCI 합산 처리

동일 INCI가 여러 원료에서 발생하면 **전부 합산 후 단일 항목으로 처리**한다.

```
[합산 예시]

단일 투입   Cyclopentasiloxane  15.00%
복합 전개   Cyclopentasiloxane   4.25%  ← Bentone Gel MIO에서
복합 전개   Cyclopentasiloxane   2.00%  ← Dow Corning 9040에서
──────────────────────────────────────
합산 결과   Cyclopentasiloxane  21.25%  ← 전성분 표기는 이 값
```

### 규칙 C — 정수 연산 필수 (PRECISION-ARITHMETIC 연계)

```python
# ✅ 올바른 방법
# 전개값을 소수로 더하지 말고, 정수로 변환 후 합산

inci_sum_int = {}

for ingredient in formula:
    if ingredient.is_compound:
        for component in ingredient.components:
            ratio_int = round(component.fraction * ingredient.int_value)
            # int_value = wt% × 100 (정수)
            inci_sum_int[component.inci] = \
                inci_sum_int.get(component.inci, 0) + ratio_int
    else:
        inci_sum_int[ingredient.inci] = \
            inci_sum_int.get(ingredient.inci, 0) + ingredient.int_value

# ❌ 금지: 소수점 직접 합산
# total = sum(x.wt_pct for x in ...)  → 부동소수점 오차 누적
```

## 5. 처방서 처리 절차 (Step-by-Step)

### Step 1 — 처방서 스캔: 복합성분 감지

```
각 투입 원료에 대해:
  IF 원료명이 compound-db.md에 존재
    → 복합성분으로 분류 (COMPOUND)
  ELSE IF INCI명이 단일 성분
    → 단일 원료로 분류 (SINGLE)
  ELSE
    → 미확인 원료 (UNKNOWN) → Gemini 리서치 트리거
```

### Step 2 — 복합성분 전개

```
각 COMPOUND 원료에 대해:
  int_value = round(투입wt% × 100)
  for each component in compound.components:
    component_int = round(component.fraction × int_value)
    expanded_list에 추가 (inci, component_int)
```

> [!CAUTION]
> 전개 시 **반올림 오차**로 구성 성분 합이 int_value와 ±1 차이날 수 있음  
> → 가장 큰 비율의 구성 성분에 차이값을 조정 (Largest Remainder Method)

### Step 3 — INCI 합산

```
merged = {}
for each (inci, int_value) in expanded_list + single_list:
  merged[inci] = merged.get(inci, 0) + int_value

# 밸런스(정제수) 항목은 합산 제외 (역산으로 계산)
balance_excluded = {k: v for k, v in merged.items() if k != "Aqua"}
```

### Step 4 — PRECISION-ARITHMETIC 연계

```
# PRECISION-ARITHMETIC SKILL과 동일한 역산 적용
balance_excluded_sum = sum(balance_excluded.values())
aqua_int = 10000 - balance_excluded_sum
merged["Aqua"] = aqua_int

# 3단계 검증
assert sum(merged.values()) == 10000          # 검증1: 정수합
assert sum(merged.values()) / 100 == 100.00   # 검증2: wt%합
assert merged["Aqua"] == aqua_int             # 검증3: 역산 교차
```

### Step 5 — 전성분 출력

- 함량 내림차순 정렬 (식약처 기준)
- 1% 이하 성분은 순서 무관 표기 가능
- 향료(Fragrance)는 단일 항목으로 표기

## 7. 처방서 ↔ 전성분 이중 문서 출력

COCHING AI는 두 가지 문서를 **동시에** 생성한다.

```
출력 A: 처방서 (제조용)           출력 B: 전성분 (규제용)
────────────────────────────     ──────────────────────────
투입 원료 기준                    INCI 기준
복합성분 상품명 표기              구성 INCI 전개·합산 표기
제조 순서 포함                    함량 내림차순 정렬
내부 문서                         포장재·규제 제출용
```

> [!TIP]
> 두 문서의 **총 wt% 합계는 동일하게 100.00%** 이어야 한다.  
> 불일치 시 전개 계산 오류 → 즉시 재검증 필요

## 9. INCI 합산 시 주의사항

### 9.1 향료 처리

```
향료 (Fragrance / Parfum) 는 구성 성분이 수백 종이지만
화장품법상 단일 항목으로 표기 가능

처리 방법:
  - 전개 하지 않음
  - "Fragrance" 또는 "Parfum" 으로 단일 표기
  - 알레르기 유발 향료 26종은 별도 표기 의무 (EU 규정)
```

### 9.2 물(Aqua) 포함 복합성분

일부 기능성 솔루션은 물을 포함한다.

```
예시: 나이아신아마이드 10% 수용액 (10% solution)
  투입: 20.00%
  전개:
    Niacinamide  → 20.00 × 0.10 = 2.00%
    Aqua         → 20.00 × 0.90 = 18.00%  ← 밸런스 Aqua에 합산!
```

> [!IMPORTANT]
> 복합성분에서 전개된 Aqua는 밸런스 Aqua와 **합산하지 않는다.**  
> 두 값을 별도로 관리 후 전성분 출력 시에만 합계를 표시한다.  
> 밸런스 역산은 반드시 **복합 전개 Aqua 제외 후** 실행한다.

### 9.3 동일 INCI 다른 등급 처리

```
예시:
  Dimethicone 5cSt  투입 8.00%
  Dimethicone 350cSt 투입 3.00%

전성분 표기:
  Dimethicone  11.00%  ← 등급 무관 INCI명 동일 → 합산
```
