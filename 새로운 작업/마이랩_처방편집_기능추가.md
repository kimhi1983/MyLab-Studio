## 기능 1 — AI 처방 아이디어 구성

처방명 + 제품유형 + 추가요구사항 입력 후
"스마트 처방 생성" 버튼 클릭 시 동작:

### 서버 API 추가 (server/index.js)
POST /api/formula/generate-idea
요청: { product_type, formula_name, requirements }
처리 순서:
  1. guide_cache에서 product_type 기존 처방 참조
  2. formula_name 키워드 파싱
     - "비건" → ingredient_type NOT IN ('animal_derived')
     - "EWG 그린" → ewg_score <= 2 필터
     - "민감성" → comedogenic_rating <= 1 우선
     - "방부제프리" → ingredient_type != 'preservative' 제외
     - "레티놀" → retinol + 안정화 원료 자동 포함
  3. purpose_ingredient_map에서
     REQUIRED 원료 → 반드시 포함
     FORBIDDEN 원료 → 자동 제외
  4. ingredient_master 물성 기반 조합
     목표 pH 범위에 맞는 원료 선택
  5. Claude SSH로 최종 처방 생성
응답: { ingredients: [...], estimated_spec: { ph, viscosity } }

### UI 수정 (AiGuide.vue 또는 FormulaEdit.vue)
- 생성 완료 후 원료 배합표 자동 채워짐
- 예상 물성(pH/점도) 미리보기 표시
- "처방 적용" 버튼으로 확정

## 기능 2 — 물성 수정 시 처방 자동 조정

우측 물성 패널(pH/점도/외관)에서 값 변경 시 동작:

### 서버 API 추가 (server/index.js)
POST /api/formula/adjust-by-spec
요청: { current_ingredients, target_spec, changed_fields }
처리:
  - pH 변경 → Citric Acid/Lactic Acid 자동 추가 또는 증량
  - 점도 변경 → Carbomer/HEC 자동 조정 + Aqua 역산
  - 외관 변경(크림→젤) → 유성원료 축소 + 수성 증점제 확대
응답: { adjusted_ingredients, preview_spec, warnings }

### UI 수정
- 물성값 변경 감지 (watch)
- 변경된 항목만 API 전달
- 조정 제안 미리보기 모달 표시
  (변경 전/후 원료 비교 + 예상 물성 비교)
- "적용" / "취소" 버튼

## 기능 3 — 원료 변경 시 자동 100% 재배분

원료 배합표에서 추가/삭제/수정 발생 시 즉시 동작:

### 프론트엔드 로직 (FormulaEdit.vue)
Aqua(밸런스) 자동 역산 규칙:
  - 원료 삭제 → 삭제된 % → Aqua에 자동 흡수
  - 원료 추가 → 추가된 % → Aqua에서 차감
  - 양 수정  → 변화량(delta) → Aqua에 반영

PRECISION-ARITHMETIC 적용:
  - 모든 계산은 정수 연산 (wt% × 100)
  - 합계 반드시 10000 유지
  - 반올림 오차 → Largest Remainder Method 보정

경고 표시:
  - Aqua가 0% 미만 → "배합비 초과" 빨간 경고
  - 합계 100% 불일치 → 상단 배너 경고
  - 규제 농도 초과 원료 → 해당 행 빨간 하이라이트

물성 실시간 재계산:
  - 원료 변경마다 /api/formula/ph-check 호출
  - pH/점도/HLB 우측 패널 즉시 업데이트
