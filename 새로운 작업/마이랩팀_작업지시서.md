### [P1] 규제 모니터링 위젯 개선

**현재 문제 (이미지1 기준):**
- 성분명 자리에 CAS번호/코드값 노출 (alpha-bisabolol, 1-3-urea 등 정상 / 나머지 비정상)
- `1,2,6-Hexanetriol` 같은 성분이 "사용금지/제한 원료 목록에 없음 — allowed" 텍스트에
  "사용금지" 단어 감지되어 금지로 오표시 → **텍스트 파싱 버그**
- `pharma_prohibited` 타입 원료(의약품 성분)가 화장품 규제 목록에 혼재

**수정 내용:**

**① 상태 판단 로직 수정 (가장 중요)**
```javascript
// src/utils/regulationSource.js 또는 Regulation 위젯

// ❌ 현재 (잘못된 방식) — 텍스트에서 키워드 파싱
const status = restriction.includes('prohibited') ? '금지'
             : restriction.includes('사용금지') ? '금지'  // ← 버그 원인
             : '모니터링'

// ✅ 수정 (DB restriction_status 컬럼 직접 참조)
// 서버에서 status 필드를 별도 컬럼으로 내려주도록 API 수정 필요
// → 워크플로우팀에 /api/regulation/monitor API 요청
const statusMap = {
  'prohibited': '금지',
  'restricted': '제한',
  'allowed': '허용',
  'monitoring': '모니터링'
}
const status = statusMap[item.restriction_status] ?? '모니터링'
```

**② pharma_prohibited 필터 적용**
```javascript
// 규제 모니터링에서 의약품/금지물질 제외
// API 요청 시 파라미터 추가 또는 프론트 필터
const filtered = items.filter(item =>
  item.ingredient_type !== 'pharma_prohibited' &&
  item.ingredient_type !== 'extract'
)
```

**③ ingredient_type 배지 추가**
```vue
<!-- 성분명 옆에 카테고리 배지 표시 -->
<span class="type-badge" :class="item.ingredient_type">
  {{ typeLabel(item.ingredient_type) }}
</span>

<!-- typeLabel 함수 -->
const typeLabel = (type) => ({
  'preservative': '방부제',
  'uv_filter': '자외선차단',
  'surfactant': '계면활성제',
  'silicone': '실리콘',
  'peptide': '펩타이드',
  'colorant': '색소',
  // ...
}[type] ?? type)
```

**④ 성분명 정렬 개선**
- 숫자로 시작하는 성분 (1,2-Hexanediol 등) → 알파벳 순 정렬 시 맨 위로 올라오는 문제
- 정렬 기준: `ingredient_type` 먼저, 그 다음 `inci_name` 알파벳 순

### [P1] 성분DB 페이지 개선 (IngredientDb.vue)

**현재 상태 (이미지2 기준):**
- INCI NAME + 한글명 + EWG + 규제상태(KR/EU) + 최대농도 표시 중
- EWG 컬럼이 모두 `-` (미수집)
- ingredient_type(카테고리) 컬럼 없음
- 물성 정보(pH, 점도 등) 없음
- 총 25,997건 — 현재 DB와 일치

**추가할 컬럼 및 기능:**

**① 카테고리 필터 추가**
```vue
<!-- 상단 필터 바 -->
<div class="filter-bar">
  <select v-model="selectedType" @change="fetchIngredients">
    <option value="">전체 카테고리</option>
    <option value="silicone">실리콘류</option>
    <option value="emulsifier">유화제</option>
    <option value="preservative">방부제</option>
    <option value="uv_filter">자외선차단제</option>
    <option value="peptide">펩타이드류</option>
    <option value="surfactant">계면활성제</option>
    <option value="humectant_active">보습/활성</option>
    <option value="extract">추출물류</option>
    <!-- 44개 카테고리 전체 -->
  </select>

  <select v-model="selectedRegulation">
    <option value="">규제 전체</option>
    <option value="허용">허용</option>
    <option value="제한">제한</option>
    <option value="금지">금지</option>
  </select>
</div>
```

**② 테이블 컬럼 확장**
```
현재:  INCI NAME | 한글명 | EWG | 규제상태 | 최대농도
개선:  INCI NAME | 한글명 | 카테고리 | EWG | pH범위 | 사용농도 | 규제(KR/EU) | 최대농도
```

**③ 상세 모달 추가**
성분 클릭 시 슬라이드 패널 또는 모달:
```
┌─────────────────────────────────────┐
│ Niacinamide (나이아신아마이드)         │
│ 카테고리: vitamin / biomimetic_active │

├─────────────────────────────────────┤
│ 기능                                 │
│  INCI 기능: Skin Conditioning        │
│  사용농도: 2 ~ 5%                   │
│  피부타입: 모든 피부                 │
├─────────────────────────────────────┤
│ 안전                                 │
│  EWG 점수: 2                        │
│  코메도제닉: 0                       │
│  IFRA 제한: 없음                     │
```

**④ EWG 점수 시각화**
```vue
<!-- EWG 점수 색상 배지 -->
<span class="ewg-badge" :class="ewgClass(item.ewg_score)">
  {{ item.ewg_score ?? '-' }}
</span>

const ewgClass = (score) => {
  if (!score) return 'unknown'
  if (score <= 2) return 'safe'      // 녹색
  if (score <= 6) return 'moderate'  // 노란색
  return 'danger'                     // 빨간색
}
```

**⑤ pharma_prohibited 별도 섹션**
- 현재 성분DB에 의약품 성분이 섞여 있음
- 기본 목록에서 `pharma_prohibited` 제외
- 별도 탭 또는 "금지물질 보기" 토글로 분리

## 워크플로우팀에 요청할 것

1. `/api/regulation/monitor` — restriction_status 별도 컬럼으로 반환
2. `/api/ingredients/db` — ingredient_type, ph_min, ph_max, ewg_score 포함
3. `/api/ingredients/search` — 원료 사전 검색
4. `/api/compounds/list` — 복합원료 목록
5. `/api/formula/expand-inci` — 전성분 전개 API
6. `/api/formula/ph-check` — pH 충돌 감지
